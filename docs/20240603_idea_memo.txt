okamoちゃんねるの3人のキャラを使ったsast兼自動パッチシステム。
コードレビュー、issue登録、プルリク、ソース修正をさせたい。

channelのキャラは利用するが、新たなシステムを作る

aiがいじれるのは指定ブランチのみ

必要な権限をもつgithub patを用意

aiはgithub mcp で指定ブランチ（例示　for_aiブランチ）を自由に操作

okamoちゃんねると同様にfargateタスクとして週１で自動起動。システム開発の際はokamoちゃんねるのソースとインフラ構成を参考にする。

github actionによる自動テストや自動ビルドは対象外（このシステムはfor_aiブランチを自律的に改善していき、レビューをつみあけていくが、それをどう評価してどう活用するかは、利用者に委ねる）

3人のキャラクターは残すが対象リポは okamo作とはかぎらないので、タスク起動時にリポジトリ作者の自己紹介をいれる


ワークフローは1→2→3

1、普通のクロードがmainブランチをfor_aiブランチにマージして、このブランチを最新か。レポジトリ構成や過去のai自動修正をサマリー（2の3人へのへの引き継ぎ用）


2、以下3人にissue→for_aiから修正用ブランチ作成→プルリクあげるように依頼

一貫して口調はキャラの口調を通す

issueはできないものも含めて指摘

whyをしっかり書くのがポイント

プルリクは修正できるものだけ



●エンジニアクロード（claude-sonnet-5を使う
→バグやセキュリティ脆弱性を指摘

●税理士GPT（gpt-5.4を使う
→無駄（デッドコードや効率の悪いコードやコスト効率の悪い設計を指摘）

●子育てママgemini（最新で安価なgemini-3.5-flashを使う
→思いやり不足（画面上の説明不足、ソースコメントの不足）を、指摘


3、まとめ役にクロードが、プルリクをレビュー、修正した場合のメリットとデメリットのトレードオフを冷静に判断し、okであえば、whyコメントをそえてマージ。ダメな場合は却下理由whyをそえて却下

まとめ役としてissueの更新を、実施

まとめ役として、
3人のissueとプルリクのコメントを元に
channel/日付/フォルダ（なければ作成）にその日のokamoちゃんねる風mdレポートを作成
